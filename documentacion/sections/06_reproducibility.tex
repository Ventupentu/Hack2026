\section{Reproducibilidad}

\subsection{Entorno}

Dependencias base en \texttt{requirements.txt}: \texttt{pandas}, \texttt{numpy}, \texttt{pillow}, \texttt{tqdm}, \texttt{requests}, \texttt{hydra-core}, \texttt{torch}, \texttt{torchvision}.

Dependencias adicionales para pipelines avanzados:
\begin{itemize}
  \item \texttt{open\_clip\_torch} (backend OpenCLIP).
  \item \texttt{ultralyticsplus} (deteccion YOLO clothing).
  \item \texttt{faiss} (opcional en \texttt{src.new\_infer.py}).
\end{itemize}

\subsection{Determinismo}

El codigo fija semillas de \texttt{random}, \texttt{numpy} y \texttt{torch}. \texttt{offline\_augment.py} usa semillas estables por hash de \texttt{asset\_id}+indice para reproducir exactamente las mismas vistas sinteticas.

\subsection{Configuracion Hydra}

La configuracion esta centralizada en:
\begin{itemize}
  \item \texttt{config/config.yaml}: hiperparametros de train/infer.
  \item \texttt{config/files/file.yaml}: rutas de datos, imagenes y caches.
\end{itemize}

Cada run genera un directorio de salida versionado por fecha/hora, lo que permite comparar variantes sin gestionar rutas manualmente.

\subsection{Contrato de entrega}

El CSV de submission debe contener:
\begin{itemize}
  \item \texttt{bundle\_asset\_id}
  \item \texttt{product\_asset\_id}
\end{itemize}

con un maximo de 15 filas por bundle y orden por confianza del modelo.
