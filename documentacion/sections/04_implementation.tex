\section{Detalles de Implementacion}

\subsection{Configuracion y Entrypoints}

El proyecto usa Hydra con dataclasses tipadas (\texttt{src/config.py}). Entrypoints principales:
\begin{itemize}
  \item \texttt{python -m src.train}: entrenamiento OpenCLIP.
  \item \texttt{python -m src.infer}: inferencia principal (salida en directorio Hydra).
  \item \texttt{python -m src.new\_infer}: inferencia v8 (salida en \texttt{outputs/}).
  \item \texttt{python -m src.infer\_phase1}: inferencia con seccion + zero-shot.
  \item \texttt{python -m src.infer\_top1}: inferencia top-1 por crop.
  \item \texttt{python -m src.infer\_openclip}: inferencia simple por argparse.
\end{itemize}

\subsection{Notas Operativas Importantes}

\begin{enumerate}
  \item \texttt{src/train.py} espera por defecto \texttt{data/product\_dataset\_with\_gender.csv} como manifiesto de productos.
  \item Si CUDA no esta disponible y se solicita \texttt{device=cuda}, el codigo hace fallback a CPU en varios entrypoints.
  \item La dependencia de deteccion YOLO es opcional en inferencia, pero necesaria para variantes guiadas por boxes.
\end{enumerate}
