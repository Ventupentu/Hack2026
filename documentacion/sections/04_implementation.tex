\section{Detalles de Implementacion}

\subsection{Configuracion y Puntos de Entrada}

El proyecto usa Hydra~\cite{yadan2019hydra} con dataclasses tipadas (\texttt{src/config.py}). Los principales puntos de entrada son:
\begin{itemize}
  \item \texttt{python -m src.train}: entrenamiento OpenCLIP.
  \item \texttt{python -m src.infer}: inferencia baseline preentrenada + particion de validacion local.
  \item \texttt{python -m src.infer\_openclip}: inferencia desde un checkpoint entrenado de OpenCLIP.
\end{itemize}

Los parametros principales de ejecucion incluyen batch size, epocas, learning rate, AMP, acumulacion de gradiente, cortes de recall y ajustes opcionales de deteccion (\texttt{bbox\_*}).

\subsection{Aspectos Clave del Loop de Entrenamiento}

La implementacion de entrenamiento OpenCLIP incluye:
\begin{enumerate}
  \item Resolucion dinamica de dispositivo (CPU/CUDA) y DataParallel multi-GPU opcional.
  \item Scheduler de learning rate coseno con 10\% de warmup.
  \item Precision mixta opcional via \texttt{torch.cuda.amp}.
  \item Logging periodico, validacion por epoca, anexado de metricas en JSONL y checkpoints (\texttt{epoch\_X.pt}, \texttt{best.pt}).
\end{enumerate}

\subsection{Logica de Inferencia y Fallback}

Ambos scripts de inferencia generan una fila por producto predicho y limitan la salida a 15 por bundle. Si no hay embeddings de bundle disponibles (por ejemplo, imagen ausente), se usan como fallback productos populares obtenidos de enlaces de train.
