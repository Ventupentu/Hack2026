\section{Metodologia}

\subsection{Formulacion del Problema}

Para cada bundle $b$, se ordena un conjunto grande de productos $\{p_i\}_{i=1}^{N}$ y se devuelven hasta 15 IDs unicos. El entrenamiento usa enlaces positivos de \texttt{bundles\_product\_match\_train.csv}.

\subsection{Representacion Multimodal con OpenCLIP}

El encoder base es Marqo Fashion SigLIP. Para productos, el sistema puede fusionar embedding visual y embedding textual de \texttt{product\_description}; para bundles se usa principalmente embedding visual de imagen/crops.

La similitud se calcula con coseno (producto punto tras normalizacion L2):
\begin{equation}
\mathrm{sim}(\mathbf{q}, \mathbf{P}) = \mathbf{q}\mathbf{P}^{\top}
\end{equation}

\subsection{Entrenamiento Contrastivo}

\texttt{src/models/retrieval\_openclip.py} optimiza una perdida contrastiva bidireccional (bundle$\leftrightarrow$product) con positivos multiples por bundle dentro del batch, scheduler coseno con warmup y AMP opcional.

\subsection{Inferencia Guiada por Deteccion}

Cuando \texttt{params.use\_bundle\_boxes=true}, se detectan regiones de ropa con YOLO y se hace retrieval por crop. Si no hay detecciones, se usa fallback de imagen completa. La cache de cajas se guarda en JSON para reutilizacion entre corridas.

\subsection{Post-procesado}

Segun script/config, se aplican:
\begin{itemize}
  \item filtro de genero (bundle section vs. \texttt{product\_dataset\_with\_gender.csv});
  \item diversidad por categoria (maximo por \texttt{product\_description});
  \item umbral minimo de score;
  \item reranking por cercania temporal (timestamp \texttt{ts} en URLs).
\end{itemize}

\subsection{Variantes de Inferencia}

El repositorio mantiene varias estrategias:
\begin{itemize}
  \item \texttt{src.infer}: pipeline principal OpenCLIP+YOLO+filtros.
  \item \texttt{src.new\_infer}: per-crop retrieval con TTA y FAISS opcional.
  \item \texttt{src.infer\_phase1}: filtrado por seccion + zero-shot de categoria por crop.
  \item \texttt{src.infer\_top1}: top-1 por crop para predicciones mas compactas.
  \item \texttt{src.infer\_openclip}: version simple por argparse.
\end{itemize}

\subsection{Metricas}

\end{equation}

\section{Methodology}

\subsection{Problem Formulation}

For each bundle query $b$, the objective is to rank product candidates $\{p_i\}_{i=1}^{N}$ and output up to 15 unique product IDs with highest relevance. Training uses known positive links from \texttt{bundles\_product\_match\_train.csv}.

\subsection{Baseline Retrieval (\texttt{src/infer.py})}

The baseline uses pretrained torchvision backbones (\texttt{resnet18}, \texttt{resnet50}, \texttt{efficientnet\_b0}) to encode bundle and product images. Embeddings are L2-normalized and cosine similarity is computed by dot product.

For each query embedding $\mathbf{q}$ and product matrix $\mathbf{P}$:
\begin{equation}
\mathrm{sim}(\mathbf{q}, \mathbf{P}) = \mathbf{q}\mathbf{P}^{\top}
\end{equation}
Top-$K$ retrieval is implemented via partial sorting (\texttt{np.argpartition}) for speed.

\subsection{OpenCLIP Multimodal Retrieval}

The advanced backend uses Marqo Fashion SigLIP~\cite{zhai2023siglip,radford2021clip}. Product embeddings can fuse image and text representations (product description tokens), while bundle queries are image-based.

Training uses symmetric bidirectional contrastive loss:
\begin{equation}
\mathcal{L} = \frac{1}{2}\Big[
\mathrm{CE}\big((\mathbf{Q}\mathbf{P}^{\top})/\tau,\mathbf{y}\big) +
\mathrm{CE}\big((\mathbf{P}\mathbf{Q}^{\top})/\tau,\mathbf{y}\big)
\Big]
\end{equation}
where $\tau$ is temperature and $\mathbf{y}=[0,\ldots,B-1]$ are in-batch positive indices.

\subsection{Detection-Guided Bundle Encoding}

When enabled (\texttt{params.use\_bundle\_boxes=true}), bundle images are processed with YOLO clothing detection~\cite{jocher2023yolo}. Each detected region is encoded, and region embeddings are averaged per bundle:
\begin{equation}
\mathbf{e}_b = \mathrm{norm}\left(\frac{1}{M_b}\sum_{m=1}^{M_b}\mathbf{e}_{b,m}\right)
\end{equation}
If no boxes are detected, a full-image fallback is used. Detection results are cached in JSON for reuse across runs.

\subsection{Validation Metrics}

Validation follows multi-label retrieval with Hit@K and Recall@K (implemented in \texttt{src/utils/metrics.py}). Given ground-truth set $G_b$ and top-$K$ predictions $\hat{G}_b^K$:
\begin{equation}
\mathrm{Recall@K}(b)=\frac{|G_b \cap \hat{G}_b^K|}{|G_b|}
\end{equation}
The reported metric is the mean across evaluated bundles.
